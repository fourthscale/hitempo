# Sprint 02.5 — UI shell

A short, design-only sprint sandwiched between auth (02) and RLS (03). No new features. We replace the minimalist scaffold layout with the dark-navy sidebar + global top bar shown in the mockups (`HiTempo-Mockups.html`), pose the design tokens, and bring the dashboard up to mockup parity. Future CRUD sprints (04+) inherit the polished shell instead of retrofitting visuals onto plain pages.

## Why now

- The auth flow is in place but the visual baseline (slate sidebar, no top bar, no real typography hierarchy) is far below the mockup standard. Continuing to build pages on it means refactoring every screen later.
- The mockup is fictional but precise enough to extract a real token set. Better to encode it once than to re-derive it sprint-by-sprint.
- We **do not** design every screen here. We design the shell + dashboard + dependency-free placeholders. Page-specific UI keeps landing inside its feature sprint.

## Goals

1. Design tokens encoded in CSS variables + Tailwind v4 `@theme` (sidebar bg, accent amber, primary teal, neutral grayscale, surface).
2. A serif display font (Playfair Display) loaded via `next/font/google` for H1/H2.
3. New `<Sidebar>` component: dark navy, hitempo logo with amber underline, "LÉON & GEORGE" subtitle, nav items with Lucide icons and (fake-for-now) counters, active item highlight, user info block at the bottom with circular initials avatar.
4. New `<Topbar>` component: centered search input with `⌘K` hint, notifications bell with red dot, "+ Nouvelle entreprise" primary CTA.
5. `app/(app)/layout.tsx` rewritten to compose `<Sidebar>` + `<Topbar>` + content area, with the content area on the light surface (slate-50).
6. `app/(app)/dashboard/page.tsx` rebuilt to match the mockup: "Bonjour {firstName} 👋" in serif, contextual subtitle, 4 KPI cards (one with a warning ring), "À traiter aujourd'hui" list, "Top cibles cette semaine" sidebar column, "Activité récente" placeholder.
7. Placeholder pages (`companies`, `contacts`, `tasks`, `settings`) get the new shell automatically and a simple page header (title + small i18n placeholder line).
8. Counters in the sidebar are real where possible: companies, contacts, tasks tables don't exist yet so we use static placeholders for now — clearly marked as such in the code so they're obvious to wire once the tables land.

## Out of scope

- New routes or new features.
- Real KPI data, real task list, real top targets — all data on the dashboard is hand-crafted placeholder. **Mark it visibly in code** with a `// PLACEHOLDER:` comment and ideally a small "v0" badge in the UI so we don't ship fake numbers by accident.
- Mobile layout. Sidebar is desktop-only for now (the L&G team works on laptops).
- Dark mode. Fixed light theme this sprint. Dark mode comes V1+ if it comes at all.
- Animations / micro-interactions beyond default hover/focus.

## Design tokens (the contract)

Encoded in `app/globals.css` via Tailwind v4 `@theme`:

```css
@theme {
  /* Brand */
  --color-brand-amber: #f59e0b;          /* logo underline, warning rings */
  --color-brand-teal:  #14b8a6;          /* primary CTA, active states */

  /* Sidebar (dark navy) */
  --color-sidebar:        #0f172a;       /* slate-900-ish */
  --color-sidebar-hover:  #1e293b;       /* slate-800 */
  --color-sidebar-active: #1e293b;
  --color-sidebar-fg:     #cbd5e1;       /* slate-300 */
  --color-sidebar-fg-muted:#64748b;      /* slate-500 */

  /* Surface */
  --color-surface:        #f8fafc;       /* slate-50, the content background */
  --color-surface-card:   #ffffff;
  --color-border:         #e2e8f0;       /* slate-200 */

  /* Typography */
  --font-sans:    "Geist", ui-sans-serif, system-ui, sans-serif;
  --font-serif:   "Playfair Display", Georgia, serif;
}
```

These names become Tailwind utilities automatically (`bg-sidebar`, `text-sidebar-fg`, `font-serif`, etc.).

## Implementation notes

Executed in one shot, ~1h.

**Token strategy.** Rather than fight the shadcn token names installed in sprint 01, I overrode the **values** of the existing CSS vars (`--sidebar`, `--sidebar-foreground`, `--primary`, etc.) in `app/globals.css` and added two custom brand vars (`--brand-amber`, `--brand-teal`) wired into Tailwind v4 via `@theme inline`. Result: `bg-sidebar`, `text-sidebar-foreground`, `bg-primary`, `bg-brand-amber`, `bg-brand-teal` all work as utilities without touching shadcn components or rebuilding the design token layer from scratch. The dark mode block (`.dark`) was left intact for later — we explicitly don't ship dark mode this sprint.

**Sidebar architecture.** Split into two files to keep Server/Client boundaries clean:
- `components/app/sidebar.tsx` — Server Component, takes `user` and `organization`, renders the static frame (logo, "LÉON & GEORGE" subtitle, user info block, sign-out form action).
- `components/app/sidebar-nav.tsx` — `"use client"`, takes a `NavItem[]` array, uses `usePathname()` to compute the active state with proper highlight.

Why split: `usePathname()` only works in Client Components, and the user/org data needs Server context. Passing the items array between them keeps the rendering tree obvious.

**Initials extraction.** `initialsFromEmail` handles `firstname.lastname@...` and `firstname-lastname@...` patterns, falls back to the first two chars of the local part. Good enough until we add real first/last name fields to the user (sprint TBD).

**Counters are placeholder.** Every counter and number on screen is hard-coded with a `// PLACEHOLDER:` comment in the source, citing the future sprint that will compute the real value. Sprint 04 wires `companies` and `contacts` counts, sprint 05 wires `tasks` count, sprint 06 wires the KPI cards. Searching for `PLACEHOLDER:` enumerates every cleanup point.

**i18n rich text.** The dashboard subtitle uses `t.rich("subtitle", { em: chunks => <span className="text-brand-amber">{chunks}</span> })` to color "8 actions vous attendent" amber. The message files use ICU plural inside an `<em>` tag (`{actions, plural, one {# action vous attend} other {# actions vous attendent}}`). next-intl handles plural+rich combination cleanly.

**Playfair Display.** Loaded via `next/font/google` with weights 500/600/700, exposed as `--font-playfair`. Tailwind utility `font-serif` resolves to it via the `--font-serif` token. Georgia is the local-machine fallback if the webfont fails.

**Topbar CTA points at `/companies/new`** which doesn't exist yet. Clicking it 404s today. Sprint 04 will land the route. Acceptable for a UI shell sprint.
